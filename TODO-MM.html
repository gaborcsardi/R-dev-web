Some of this idea, I started to discuss in Vienna.. This would *not* be a new argument to qnorm(), but rather a global thing in options(..), something like

	      options(dpq.precision  = 1e-10)
    which would use faster less precise algorithms when the required precision was set high (i.e., 1e-4 instead of 1e-10) Feedback? Not worth the hassle?
