Martin Maechler's TODO list

This page contains (some of) my plans for enhancements to R ... with absolutely no warranty ...

Martin Maechler maechler@r-project.org

Shorter term, maybe for R 1.1.x

  • Implement Clive Loader's dbinom(), dpois() improvements.
  • Replace qnorm()'s algorithm with AS 241.
    I've been considering an option which would allow to use the old (quite less precise, but a tad faster) version. Some of this idea, I started to discuss in Vienna..
    This would not be a new argument to qnorm(), but rather a global thing in options(..), something like 
    	      options(prob.precision = 1e-10) # 1e-10 is somewhat arbitrary.
	      options(dpq.precision  = 1e-10)
    which would use faster less precise algorithms when the required precision was set high (i.e., 1e-4 instead of 1e-10).

    Feedback? Not worth the hassle?

Of less priority, some AFTER 1.1.x

  • scatterplot3d from Uwe Ligges): A simple 3D scatterplot with somewhat orthogonal functionality to persp().
  • Running Medians for library(modreg) : We need a fast robust smoother; lowess() is not robust.
  • New print option for printing exact "0"s. Consider using "." instead of "0" in numerical matrices which makes sparse matrices much more readable.
    [I have been doing this with S-plus for years]
  • scat1d() {improved rug(); the thing I've co-authored which has now been put into Hmisc}.
  • stem() should exclude outliers and list them separately (as S does).

Even Longer Term

  • Numerical Integration.
    I want to have something as good as Quadpack. GSL -- The GNU Scientific Library seems the place to tap for these things, and I have had a first successful dyn.load() working with (part of) it.
  • Differential Equation Solvers (ODE at least), also using GSL (see above).
