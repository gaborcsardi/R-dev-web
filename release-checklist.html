Release checklist

  • Before starting release procedures:
    • Patch releases (x.y.z, z ≥ 1)
      • CRAN masters: Populate CRAN subdirectory
      • Set version to "x.y.z RC"
      • Update the NEWS and src/gnuwin32/CHANGES file headers from x.y.(z-1) Patched to x.y.z
    • Dot-releases (x.y.0):
      • Create the SVN release branch
      • Set VERSION of SVN trunk to "x.y+1.0 Under development (unstable)" (NB: This has to be done manually)
      • CRAN masters: Populate CRAN subdirectory for development version
      • Set VERSION in release branch to x.y.0 alpha (use the R-set-version script)
  • At release initiation:
    • Setup crontab for nightly builds
    • (Dot-release only) schedule transition to beta and RC status
    • Update developer page
    • Update R-project homepage
    • Notify r-devel list
  • At release time:
    • Create final build (edit and run R-build-dist script)
    • Update developer page
    • Update R-project homepage
    • Calculate md5sums
    • Notify r-announce list. Include md5sums and relevant section of NEWS.
