Release checklist

"Release procedures" are initiated when the crontab for prerelease builds is created. This will be at (or shortly before) "Grand Feature Freeze" for x.y.0 or approximately one week prior to release of x.y.z.

  • Before starting release procedures:
    • Patch releases (x.y.z, z ≥ 1)
      • CRAN masters: Populate CRAN subdirectory
      • Set version to "x.y.z RC" (or "x.y.z beta" if some time before prerelease)
      • Update the NEWS and src/gnuwin32/CHANGES file headers from x.y.(z-1) Patched to x.y.z
    • Dot-releases (x.y.0):
      • CRAN masters: Populate CRAN subdirectory for development version
      • Edit and run the R-create-branch script, which does the following.
        • Create the SVN release branch
        • Set VERSION of SVN trunk to "x.y+1.0 Under development (unstable)"
        • Set VERSION in release branch to "x.y.0 alpha"
      • Update developer page
      • Update R-project homepage
      • Notify r-devel list
  • At release initiation:
    • Setup crontab for nightly builds
    • (Dot-release only) schedule transition to beta and RC status
  • At release time:
    • Create final build (edit and run R-build-dist script)
    • Set version to "x.y.z Patched"
    • Update developer page
    • Update R-project homepage
    • Calculate md5sums
    • Notify r-announce list. Include md5sums and relevant section of NEWS.
